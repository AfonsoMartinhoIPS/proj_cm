# Mobile App Project – Overview

## Objective
Develop a complete mobile application using Flutter that demonstrates technical competence, good architecture, and solid user experience.

## Core Requirements
The application must include:

- User authentication (registration and login)
- Notifications (push or local, relevant to the app)
- Integration with at least one external API
- A remote database (e.g., Firebase Firestore)

## Application Scope
The app should be a fully functional product, not a prototype. It must include:

- Multiple screens
- Clear navigation flow
- Persistent data storage
- Clean and consistent user interface
- Proper handling of loading and error states

## Technical Expectations
- Organized and modular code structure
- Separation between UI, business logic, and data layers
- Use of appropriate Flutter patterns and state management
- Proper API handling (requests, responses, errors)

## UI/UX Expectations
- Consistent layout and styling
- Responsive design for different screen sizes
- Clear and intuitive navigation
- Visual feedback for user actions

## Team and Workflow
- Developed in a team using Git
- Consistent commits and clear version history
- Collaborative development practices

## Deliverables

### Intermediate Delivery
- App concept and target users
- Description of features
- Wireframes or mockups
- Navigation structure
- Partial UI implementation
- Initial project setup

### Final Delivery
- Fully functional application
- All requirements implemented
- Clean and structured codebase
- Working API integration and database
- Notifications implemented
- Updated and organized repository

## Evaluation Focus
- Correct functionality of all features
- Code organization and architecture
- UI/UX quality
- Performance and responsiveness
- Documentation quality
- Originality and added value

## Summary
The goal is to build a complete, well-structured mobile application that reflects both technical skill and good design practices.
